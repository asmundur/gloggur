# Bonkers Search Failure Matrix

This matrix tracks the failure families that can produce plausible-looking but ungrounded search output. The rule for every row is the same: fail loud, emit machine-readable diagnostics, and avoid returning weak semantic-only junk as if it were grounded evidence.

| Family | Trigger | Observable symptom | Current detection | Missing detection addressed here | Remediation / code |
| --- | --- | --- | --- | --- | --- |
| entrypoint/version skew | Callers use `HybridSearch.search()` or legacy projections instead of `gloggur search --json` ContextPack v2 | Payload looks valid but lacks router/debug/health fields, making bad semantic output look authoritative | ContextPack v2 migration docs existed, but legacy API calls could still look normal | Legacy entrypoints now emit `entrypoint`, `contract_version`, `warning_codes`, and deprecation notice | `legacy_search_surface`, `legacy_search_contract`, migrate to `gloggur search --json` |
| index health | Missing index metadata, embedding-profile drift, interrupted index, or advisory tool-version drift | Search runs against stale or incomplete state, or stale callers keep trusting cache contents | Resume contract existed in CLI `status/search` | Shared health snapshot now feeds `status`, CLI search, and legacy HybridSearch so real incompatibilities block semantic search while pure tool-version drift is surfaced as a warning | `missing_index_metadata`, `index_interrupted`, `embedding_profile_changed`, `tool_version_changed` |
| vector/cache divergence | Chunk rows and vector ids drift apart, or vector consistency cannot be verified | High-score semantic hits point at stale/unrelated symbols | Post-index consistency check existed but only influenced index success | Integrity markers are now persisted and surfaced to search; semantic retrieval is suppressed when vector integrity is failed or missing | `vector_metadata_mismatch`, `vector_consistency_unverifiable`, `vector_integrity_missing`, reindex |
| chunk/span integrity | Chunk line spans escape symbol bounds, overlap neighboring symbols, or chunk text is empty for non-empty symbols | Result snippets/context include unrelated symbol bodies and misleading nearby code | No explicit chunk/span validation existed | Indexing now validates chunk boundaries and fails the file/index instead of silently persisting misleading spans | `chunk_span_integrity_error`, `chunk_span_integrity_failed`, reindex after fixing parser/chunk construction |
| routing/threshold degradation | Auto mode has only weak semantic candidates below threshold | Query like `Database setup` returns unrelated OAuth code with a high-looking raw score but no lexical support | Router thresholds existed, but auto mode still merged low-quality semantic fallback | Router now suppresses semantic-only below-threshold results and emits warning payloads instead of returning junk | `ungrounded_results_suppressed`, inspect `summary.backend_thresholds` / `debug.backend_thresholds` |
| embedding/model quality | Embedding provider initializes but model quality is weak for the repository/query domain | Semantic results cluster around irrelevant concepts with high raw similarity and low lexical overlap | Backend quality scoring and thresholds | Per-backend threshold status is now exposed in summary/debug so “raw score high, threshold not met” is explicit | Check `backend_thresholds`, tune profile, reindex with the intended model |
| payload/compat projection | Compatibility adapters reshape `hits[]` into legacy `results[]` and omit provenance | Downstream tooling sees a familiar legacy shape and misses that the result was suppressed, legacy, or v2-derived | Legacy projection kept basic metadata | Projection now preserves source schema version, entrypoint, warning codes, and integrity fields | `source_schema_version`, `source_entrypoint`, `warning_codes` |

## Operational guidance

- Prefer `gloggur search <query> --json --debug-router` when validating suspicious results.
- Treat `warning_codes` as first-class signals, not optional hints.
- If `semantic_search_allowed` is `false`, do not trust semantic output until the index is rebuilt and integrity markers are present.
- If a caller still depends on `HybridSearch.search()`, treat it as transitional and budget migration work to ContextPack v2.
